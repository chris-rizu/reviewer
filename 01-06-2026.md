## 2. データインポートスクリプト修正

### 発見された問題と対応

| 問題                          | 原因                       | 対応内容                                        |
| --------------------------- | ------------------------ | ------------------------------------------- |
| `createClient()` 引数なし       | 環境変数未設定                  | `SUPABASE_URL` と `SERVICE_ROLE_KEY` を渡すよう修正 |
| `users.id` NULL制約違反         | `auth.users` を参照する外部キー制約 | ユーザーインポートをスキップ                              |
| `action_types` テーブル不在       | テーブル未作成                  | レポート項目インポートをスキップ                            |
| CSVエンコーディング問題               | Shift-JIS形式              | `iconv-lite` で変換                            |
| `customers_rank_check` 制約違反 | `SS/S` が許可されていない         | `A/B/C/D/E/Z` へマッピング                        |

### 修正対象

```text
scripts/import-data-sprint.ts
```

# 🔧 技術的ハイライト

## 解決した技術課題

### 1. Shift-JIS CSV 読み込み

* `iconv-lite` を利用
* 日本語CSVをUTF-8へ変換

### 2. Supabase Auth 外部キー制約

* `users.id` が `auth.users` を参照する構造を理解
* 認証データとの整合性を考慮した対応を実施

### 3. CHECK 制約違反

* ランク値をデータベース制約に合わせて変換

```text
SS → A
S  → B
```

（データ仕様に合わせてマッピング処理を実装）
